## 项目概况

《马书》（Harness Engineering: From Claude Code to AI Coding）是一本基于 Claude Code v2.1.88 源码逆向分析的开源书籍，系统拆解了 AI 编码 Agent 的全套技术栈。全书约 30 章，分 7 篇 + 7 个附录，中英文双语。项目本身是一个 AI 辅助写作的示范：用 Claude Code 分析 Claude Code 源码，用 spec 驱动章节产出，用 CLAUDE.md 规范写作质量。

- **技术栈**：TypeScript + React Ink + Bun（被分析对象），mdBook（书籍构建）
- **逆向方法**：源码阅读 + 行为实验 + 版本 diff 追踪
- **核心理念**："驾驭工程"（Harness Engineering）—— 模型是引擎，驾驭系统是整车

## 前三章概要

**第 1 章：AI 编码 Agent 的完整技术栈**
拆解 Claude Code 的三层架构。应用层 1,884 个 TypeScript 文件，运行时层 Bun/JavaScriptCore，外部依赖层 npm/API/MCP。核心机制：main.tsx 并行预取启动、AppState 全局状态、"On Distribution" 理念（模型运行时动态决定行为）、89 个 Feature Flag 通过 bun:bundle 的 feature() 做死代码消除。

**第 2 章：工具系统 —— 40+ 个工具作为模型的双手**
从 Tool.ts 接口契约到工具注册管线全景。buildTool() 工厂函数提供"失败关闭"默认安全值，tools.ts 实现三级过滤（编译期 Feature Flag → getAllBaseTools 组装 → getTools 运行时过滤），MCP 工具融合机制。还有工具结果大小预算和三阶段渐进渲染。

**第 3 章：Agent Loop —— 全书锚点章节**
深入 queryLoop() 约 1,730 行的状态机。10 字段 State 类型、7 种 Continue 转换、10 种 Terminal 终止原因。单次迭代 8 个阶段：上下文预处理管线（snip → microcompact → collapse → autocompact）→ 消息标准化 → API 调用 + attemptWithFallback → 流式工具执行 → 附件注入 → 多层恢复机制。

## 最值得先读的章节：第 3 章

**理由**：第 3 章被前言和 CLAUDE.md 明确定位为"全书锚点"。所有后续章节 —— 第 5 章（API 调用构建）、第 9 章（自动压缩）、第 13 章（流式响应）、第 16 章（权限检查）—— 都会引用第 3 章来定位各自在循环中的执行位置。理解 Agent Loop 就等于有了一张全书的心智地图。

## 推荐阅读路径

- **路径 B（安全工程师）**：第 1 章 → 第 3 章 → 第 16-19 章（安全与权限）

## 关键收获

1. **"驾驭工程"的核心洞察**：模型能力是上限，驾驭系统决定实际产出。好的驾驭设计应该预设组件会随模型进步而过时。
2. **Agent Loop 是心脏**：理解 queryLoop 的状态机，就理解了 AI Agent 的完整运转机制。
3. **Feature Flag 驱动演进**：89 个 Feature Flag 不仅是开关，更是产品路线图的反向工程入口。
4. **AI 辅助写作的示范**：项目本身用 spec 驱动、CLAUDE.md 规范、版本 diff 追踪的写作流程，就是一套可复用的技术写作方法论。
