# Coze 进阶：Chatflow + 知识库 + 插件

> 基础 Bot 只会按人设回复，加了 Chatflow + 知识库 + 插件，才能真正接单赚钱。

---

## 一、工作流（Workflow）详解

### 1.1 什么是工作流

工作流是 Bot 内部的**自动化处理流水线**。没有工作流时 Bot 只能靠 LLM 自由发挥，有了工作流 = 给 Bot 装了一套 SOP。

### 1.2 搭建第一个工作流（实战：每日新闻 Bot）

#### Step 1：创建工作流

扣子首页 → 资源库 → 资源 → 工作流 → 新建

输入名称：`每日精选新闻`，描述：`获取科技新闻并精选整理`

#### Step 2：添加节点

```
开始节点
  ↓
插件节点（Science_news）     ← 搜索"科技新闻"插件，拉取最新新闻
  ↓
大模型节点（精选新闻）        ← 输入提示词"从以下新闻中精选3条最重要的，100字总结每一条"
  ↓
结束节点（输出精选结果）
```

#### Step 3：设置参数

- Science_news 插件 → 参数留空使用默认
- 大模型节点 → 输入引用 `{{上游节点.输出字段}}`
- 结束节点 → 输出变量设为大模型节点的输出

#### Step 4：试运行 → 发布

点击试运行，看效果。满意后发布工作流。

### 1.3 工作流挂载到 Bot

回到 Bot 编辑页 → 编排 → 添加工作流 → 选择已发布的工作流。

在人设中描述何时触发：`当用户说"今日新闻"时，调用"每日精选新闻"工作流`。

## 二、知识库（Knowledge Base）

### 2.1 什么是知识库

把你的文档/网页/表格喂给 Bot，它就能回答你专属领域的问题。比如：
- 上传公司产品手册 → Bot 变成智能客服
- 上传法规文件 → Bot 变成法律助手
- 上传课程资料 → Bot 变成学习助教

### 2.2 配置知识库

#### 文本知识库

1. 扣子首页 → 资源库 → 知识库 → 新建
2. 上传文档（PDF/Word/TXT/Markdown）
3. 设置分段策略：
   - 自动分段：适合大多数场景
   - 自定义分段：按标题/段落手动切分
   - 推荐：每段 500-1500 字符

#### 表格知识库

适合结构化数据（产品目录、价格表、FAQ）。上传 CSV/Excel，Bot 可以精确查询。

#### 图片知识库

上传产品图片/证件模板，支持以图搜图。

### 2.3 知识库挂载

Bot 编辑页 → 编排 → 添加知识库 → 选择 → 保存。

Bot 收到问题时会优先检索知识库，找不到才用 LLM 通用知识回答。

### 2.4 知识库优化技巧

| 问题 | 方案 |
|------|------|
| 检索不准确 | 缩小分段，每段加标题 |
| 回答太泛 | 提高检索匹配度（降低 top_k 到 3-5） |
| 漏答 | 确保文档覆盖问题，调整分段边界 |
| 答案冗长 | 在人设中限制输出长度 |

## 三、插件（Plugins）生态

### 3.1 插件类型

| 类型 | 用途 | 示例 |
|------|------|------|
| 搜索 | 联网搜索 | 必应搜索、Google 搜索 |
| 阅读 | 读取链接内容 | 头条搜索、网页抓取 |
| 图片 | 生成/识别图片 | 图片理解、DALL·E |
| 工具 | 实用功能 | 思维导图、PDF 生成 |
| 生活 | 天气、地图等 | 墨迹天气、高德地图 |
| 知识 | 专业知识库 | 学术搜索、法律库 |

### 3.2 插件选择策略

```
需要联网查资料？    → 必应搜索 / 头条搜索
需要读网页内容？    → 网页抓取
需要生成图片？      → 图片生成
需要专业知识？      → 知识库（比插件更准）
需要复杂计算？      → 代码节点（写 Python）
都没有？            → 自建插件（API 对接）
```

### 3.3 自建插件

当你需要的 API 不在 Coze 插件商店时：

1. 扣子首页 → 资源库 → 插件 → 新建
2. 填写 API 地址、请求方式、参数
3. 测试 → 发布
4. 像官方插件一样使用

## 四、Chatflow 模式（Coze 3.0+ 核心）

### 4.1 Chatflow vs 自由对话

| | 自由对话 | Chatflow |
|------|------|------|
| 控制力 | 依赖 LLM 理解 | 精确控制每个分支 |
| 适合 | 闲聊、简单问答 | 多步骤业务流程 |
| 节点 | 无 | 条件判断/循环/意图识别 |

### 4.2 Chatflow 典型节点

```
开始节点
  ↓
意图识别节点  ← 判断用户意图（退款？咨询？购买？）
  ↓
  分支1（退款）→ 查询订单 → 判断状态 → 自动退款 / 转人工
  分支2（咨询）→ 检索知识库 → 生成回答
  分支3（购买）→ 推荐商品 → 生成下单链接
  ↓
结束节点
```

### 4.3 什么时候用 Chatflow

- 业务有明确分支逻辑（退款/咨询/投诉）
- 需要调用多个 API 串联
- 需要精确的输出格式
- 需要记录每个节点的结果用于后续分析

## 五、实战：做一个「智能客服」Bot

综合运用 Chatflow + 知识库 + 插件：

```
1. 知识库：上传产品手册、退换货政策、FAQ
2. 插件：添加"订单查询"插件（对接你的数据库 API）
3. Chatflow：
   ├─ 意图识别：区分咨询 / 退款 / 投诉
   ├─ 咨询分支：检索知识库回答
   ├─ 退款分支：查订单 → 判断是否可退 → 生成退款链接
   └─ 投诉分支：收集信息 → 生成工单 → 发送到飞书
4. 发布：微信服务号 / 飞书 / 网页嵌入
```
